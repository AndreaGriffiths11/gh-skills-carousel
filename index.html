The SKILL.md pain

"I've been using SKILL.md files for months. Custom instructions for different tasks."

"The annoying part has always been distribution — copying files manually like it's 2012."

GitHub shipped a fix

Last week: gh skill

A new GitHub CLI command that does for agent skills what npm did for packages.

What are agent skills

SKILL.md files — folders of instructions, scripts, and resources that tell an AI agent how to handle a specific task.

Works across GitHub Copilot, Claude Code, Cursor, Codex, and Gemini CLI.

How it works

gh skill install github/awesome-copilot documentation-writer

Pin to a version:

gh skill install ... --pin v1.2.0

Or pin to a commit for full reproducibility.

The supply chain part

Agent skills are instructions that shape what AI does inside your codebase.

A silently modified skill is a real attack surface.

Same as a tampered npm package, just newer.

Built-in security

Provenance metadata written into SKILL.md frontmatter

Source repo, ref, and git tree SHA

Immutable releases mean content can't be altered after publication

Why this matters

The SKILL.md pattern has been spreading quietly for months.

What was missing was tooling. No dependency graph, no version history, no integrity check.

gh skill is the package manager layer the ecosystem needed.

Try it now

gh extension upgrade --all

gh skill install github/awesome-copilot documentation-writer

Look at the publish workflow before the ecosystem gets crowded.
